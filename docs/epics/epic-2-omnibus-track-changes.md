Instead: **curated source file** `data/source/amendments/pe-cons-30-26.json`, transcribed per instruction with page-image cross-check (precedent: `~/ai-omnibus-2025-0359` transcription skills).

**AGENTS.md carve-out required** (approved): golden rule 2 stays for the base corpus (parser-only from EUR-Lex HTML); amendment data is a separate curated source with documented review procedure, **scheduled for replacement** by a deterministic parse of the CELEX 32026R… HTML once in OJ. Pipeline boundary that makes the swap cheap: everything downstream reads `data/generated/amendments.json`; only the producer changes.

## Data model

Source schema (`data/source/amendments/pe-cons-30-26.json`), one entry per (sub-)instruction:

```ts
interface Amendment {
  seq: number; sub?: string;                     // "2" + "a" = instruction 2)a)
  target: { article?: string; annex?: string };  // article: "2", "4bis", "75quater"
  operation: "replace" | "insert" | "add" | "delete";
  scope: { anchor?: string; wholeArticle?: boolean; description: string };
    // anchor reuses existing convention: "lid-2", "lid-2-g", "punt-14"
  newContent?: ContentNode[];                    // reuses existing node model
  newArticle?: { slug: string; title: string; insertAfter: number;
                 paragraphs: ArticleParagraph[] };
  newAnnex?: { roman: "XIV"; title: string; content: ContentNode[] };
  note?: string;                                 // e.g. staggered application dates — text only, no schema
}
```

Generated by `scripts/parse-amendments.ts` (wired into build chain):
- `data/generated/amendments.json` — normalized list + index `{ byArticle, byAnnex, orderedTargets }`.
- `data/generated/amendment-diffs.json` — per affected article, per paragraph:

```ts
type DiffSegment = { op: "eq" | "ins" | "del"; text: string };
interface ParagraphDiff { anchor: string; status: "modified"|"inserted"|"deleted"|"unchanged";
                          segments?: DiffSegment[]; newContent?: ContentNode[]; seq: number[] }
```

Word-level diff via `diff` npm devDep (jsdiff `diffWordsWithSpace`) over flattened old/new paragraph text (markers included) — don't hand-roll LCS. Accepted trade-off: diff view renders flat prose with `<ins>`/`<del>` (nested list layout lost in changed lids); the clean "nieuwe tekst" view keeps full structure via `newContent`.

**Diff invariant (the strong verify check)**: `concat(eq+del) === oldFlat && concat(eq+ins) === newFlat`, exactly.
